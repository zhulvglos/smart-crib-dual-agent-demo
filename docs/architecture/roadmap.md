# 婴儿床监护系统 - 增强建议

## 本次修改内容

### 1. 修复三区分类逻辑

**问题：** 之前使用 `is_near_crib_edge(threshold=0.1)` 判断危险，导致检测点只要在黄框（crib_contour）内部、距离边缘 10% max_dim 以内就触发 DANGEROUS。这太敏感了——婴儿明明还在床内就被判为危险。

**修复：** 改为严格的多边形包含判断：
- **SAFE**（绿色）：所有关键检查点都在 safe_contour（绿框）内部
- **WARNING**（黄色）：所有关键检查点在 crib_contour（黄框）内部，但至少有一个不在 safe_contour 内部
- **DANGEROUS_ACTION**（红色）：任一关键检查点在 crib_contour（黄框）外部

### 2. 增加多点安全检查

之前只使用一个 trigger_point（默认 bbox_right）进行判断，容易因检测框抖动而误判。

现在综合使用以下检查点：
1. **trigger_point**（如 bbox_right / bbox_left）— 保持与配置兼容
2. **center**（检测框中心）— 稳定参考
3. **bottom_center**（检测框底部中心）— 婴儿的脚/身体下部，最能反映是否爬出床
4. **YOLO-pose 关键点**（如果可用）— 头部、肩膀、臀部、脚踝

任一检查点越界即触发对应状态。

### 3. YOLO-pose 姿态关键点支持

扩展了 `Detection` 类和 `run_detector` 函数，当使用 YOLO-pose 模型（如 `yolo11n-pose.pt`）时，会自动提取人体关键点用于更精确的判断。

提取的关键点包括：
- **头部**（鼻子）：判断婴儿头部是否探出床栏
- **肩膀中心**：判断上半身位置
- **臀部中心**：判断躯干位置
- **最低脚踝**：判断脚是否已经伸出床外

## 进一步提升婴儿检测准确性的建议

### 短期可执行（1-2 周）

#### 1. 切换到 YOLO-pose 模型

**操作：** 将 `config/demo_detector_config.json` 中的模型从 `yolo11n.pt` 改为 `yolo11n-pose.pt`。

**好处：**
- 获得 17 个人体关键点，可精确知道婴儿的头部、手脚位置
- 可以判断婴儿姿态：趴着（俯卧）、仰睡、侧睡、站立、攀爬
- 不再依赖 bbox 边缘，直接用鼻子/脚踝位置判断越界

```bash
# 自动下载
python demo_danger_action_detector.py --model yolo11n-pose.pt
```

#### 2. 添加婴儿专用检测器

当前使用通用 "person" 检测，会把成人也识别为目标。

**方案 A：数据集微调**
- 收集婴儿床场景下的婴儿图片（500-1000张）
- 在 COCO 预训练模型基础上微调，训练 "baby" 类别
- 使用工具：[LabelImg](https://github.com/tzutalin/labelImg) 标注，Ultralytics 训练

**方案 B：级联过滤**
- 先用 person 检测器找出所有人体
- 再用婴儿特征规则过滤：
  - 面积比例（婴儿通常占画面 10%-40%）
  - 长宽比（婴儿爬行时 bbox 较宽，站立时较高）
  - 配合姿态关键点：婴儿身高通常较短（肩到脚踝距离小）

#### 3. 添加人脸检测确认

引入专门的婴儿人脸检测模型（如 YuNet 或 MediaPipe Face Detection）：
- 确认目标确实是婴儿（人脸特征：比例、大小）
- 检测婴儿面部朝向（面朝下可能有窒息风险）
- 检测婴儿是否清醒（眼睛开合状态）

```python
# MediaPipe 人脸检测示例
import mediapipe as mp
face_detection = mp.solutions.face_detection.FaceDetection(min_detection_confidence=0.5)
```

#### 4. 优化目标选择逻辑

当前 `select_target` 选择最大的人体框。在婴儿床场景中可能有多个目标（婴儿 + 成人）。

**改进：**
- 优先选择与上一帧位置最接近的目标（已部分实现）
- 添加婴儿特征打分：面积适中 + 在床区域内 + 有婴儿姿态特征
- 如果检测到多个 "person"，选择那个 keypoints 数量最多/最完整的

### 中期优化（1-2 个月）

#### 5. 姿态分析（Pose Analysis）

基于 YOLO-pose 的关键点，分析婴儿的危险姿态：

| 姿态 | 关键点特征 | 风险等级 |
|------|-----------|---------|
| 仰睡 | 鼻子朝上，肩膀和臀部水平 | SAFE |
| 趴睡 | 鼻子朝下，肩膀高于臀部 | WARNING（窒息风险） |
| 侧睡 | 肩膀垂直，臀部倾斜 | SAFE |
| 站立 | 脚踝到鼻子距离接近床高度 | WARNING（可能跌落） |
| 攀爬 | 一只手或脚超出床轮廓 | DANGEROUS |
| 面部遮挡 | 手/枕头覆盖鼻子区域 | DANGEROUS |

实现方式：计算关键点之间的角度和距离比例，使用规则或轻量级分类器。

#### 6. 时序行为分析

单帧检测容易误判。引入时序模型分析婴儿的行为模式：

**简单方案（帧级状态机）：**
- 记录最近 N 帧的状态序列
- 检测异常模式：如从 SAFE 快速移动到 DANGEROUS（婴儿突然翻身）
- 增加 "犹豫期"：WARNING 持续超过一定时间才升级为 DANGEROUS

**进阶方案（LSTM/GRU）：**
- 输入：连续 30 帧的关键点坐标 + bbox 信息
- 输出：行为分类（正常睡眠 / 翻身中 / 攀爬中 / 跌落风险）
- 训练数据：标注好的婴儿床视频片段

#### 7. 多模态融合（视频 + 音频）

当前只有视频检测。增加音频分析：
- 婴儿哭声检测：使用音频分类模型（如 YAMNet、Wav2Vec）
- 哭声 + 攀爬姿态 = 高风险
- 哭声 + 仰睡姿态 = 可能是饿了/需要换尿布（非危险）

#### 8. 深度估计（如果硬件支持）

如果有深度摄像头（如 RealSense）或双目摄像头：
- 估计婴儿到床栏杆的实际距离
- 3D 空间中判断婴儿是否真正越界
- 检测婴儿是否被被子覆盖（深度异常）

### 长期规划（3-6 个月）

#### 9. 端到端婴儿危险检测模型

训练一个专门的 "婴儿危险行为检测" 模型：
- 基于 YOLOv8 或 RT-DETR 架构
- 输出：bbox + 危险分数 + 危险类型（攀爬/跌落/窒息/被覆盖）
- 数据集：收集大量婴儿床场景视频，标注危险事件

#### 10. 边缘部署优化

当前在 CPU 上运行，帧率可能不高。

**优化方向：**
- 使用 ONNX / TensorRT / OpenVINO 加速推理
- 量化模型（INT8）减少计算量
- 降低推理分辨率（320x320 代替 640x640）
- 只在感兴趣区域（ROI）内运行检测
